100 Days of Code: Day 1 - GitHub

3/25/2018

Today was my first day of the 100 Days of Code challenge. I’m taking this challenge as a way to stay accountable in my quest for daily learning habits. I’d like to spend at least one hour a day coding or learning something new. My focus for day 1 of 100 days of code was Github. I’d like to use Github for many things, but my main focus for now was setting up my portfolio website.

So, what is GitHub? Before today, I was not so sure. With the many resources online, I found that Github is actually pretty cool. It acts as a social network for developers. We upload our projects to Github and anyone can see our code and suggest changes be made to it. It can be a very effective collaboration tool. Git is version control software and allows for multiple users to work on a project at once without losing important code to changes made by someone else. Every change to the project is saved and can be reverted back.

With the help of a Readwrite.com article called "GitHub for Beginners: Don't Get Scared, Get Started", I was able to upload my website utilizing Git on the command line.

With the help of Tyler Fisher's article titled, "Steps for Hosting a Website on Github", I was able to make the site live on the Github platform, just like a hosting provider. The key in doing this was naming the repository username.github.io, where username is your github username.

One problem I did have was getting my .php contact form to work. Github allows for one static website, but it does not support php or any other server side languages. So, what to do? A helpful user on Stack Overflow suggested Formspree for contact forms on static sites. Let me tell you, Formspree is such an easy to implement solution to my problem. It works great and is free! Definitely recommend!

My trophy for today is a deployed website on taylerjohannes.github.io
